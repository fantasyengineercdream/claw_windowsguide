# 给没用过 GitHub 的小白用户怎么交付

## 结论

可以发 GitHub 链接，但不要只发仓库首页然后让用户自己猜。

更稳的做法是：

1. 发 ZIP 下载链接
2. 发一句非常短的说明
3. 发一张安装器截图
4. 如有条件，再发一个 30-60 秒录屏

## 最推荐的交付方式

顺序建议：

1. 直接发 `ZIP`
2. 再发一句话说明
3. 再发截图

推荐说明话术：

`这是给 Windows 用的 OpenClaw 飞书安装包。先按飞书官方文档准备好飞书应用，再把 OpenClaw 模型配好，然后双击 install.cmd，最后按界面做配对。`

## 如果必须走 GitHub

请优先发：

- 仓库首页
- 或 ZIP 直链

但一定要同时告诉用户：

1. 不要先看代码
2. 先看 README 顶部
3. 先按顺序准备飞书应用和 OpenClaw 模型
4. 再双击 `install.cmd`

## 小白最容易卡住的点

1. 不知道飞书 `App ID / App Secret` 去哪里拿
2. 以为双击安装器就能直接聊天
3. 不知道模型必须先配好
4. 不知道最后还要做一次配对

## 最低交付包建议

至少包含：

- `openclaw-windows-hardlock.zip`
- `START_HERE_CN.txt`
- `README.md`
- 安装器截图

## 不要这样交付

不建议：

- 只发 GitHub 首页
- 只发仓库代码页
- 只说“自己看 README”

小白用户最需要的是明确顺序，而不是更多信息。
