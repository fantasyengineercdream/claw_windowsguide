# 给从没用过 GitHub 的小白用户怎么交付

## 结论

不建议只把 GitHub 仓库首页丢给小白用户。

原因很简单：

- GitHub 首页对技术用户友好，对小白不友好。
- 用户会先被目录、提交记录、脚本名和文档结构吓住。
- 很多人甚至不知道“Code”“Releases”“Download ZIP”分别是什么意思。

## 最推荐的交付方式

推荐按这个顺序交付：

1. 直接发 `Release ZIP`。
2. 同时发一句非常短的话术。
3. 再附一张安装器截图。
4. 如有条件，再发一个 30-60 秒录屏。

## 推荐话术

你可以直接发这段：

`这是给 Windows 用的一键安装包。你下载并解压后，直接双击 install.cmd，然后按界面提示填飞书 App ID 和 Secret 就可以。`

## 对小白最友好的交付形态

优先级从高到低：

1. 网盘/IM 直接发 ZIP 包
2. GitHub Releases 页面
3. GitHub 仓库首页

## 如果必须走 GitHub

不要发仓库首页，优先发：

- `Releases` 页面
- 具体版本的 ZIP 下载链接

并且同时告诉对方：

- “不要先看代码”
- “先下载 ZIP”
- “解压后双击 install.cmd”

## 最低配交付包应该包含什么

至少包含：

- `openclaw-windows-hardlock.zip`
- `START_HERE_CN.txt`
- `README.md`
- 安装器截图

## 你对外最稳的交付姿势

如果对方完全不懂技术，最佳做法是：

1. 发 ZIP
2. 发一句话说明
3. 发截图
4. 发你自己的飞书联系方式或反馈入口

这样比直接发 GitHub 仓库，转化率和成功率都高得多。
