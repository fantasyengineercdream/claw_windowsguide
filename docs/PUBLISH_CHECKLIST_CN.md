# 发布前检查清单

如果你准备把这个项目发给新手用户，建议先检查下面这些点。

## 1. 用户需要什么

你要提前告诉用户：

- 需要 `飞书 App ID`
- 需要 `飞书 App Secret`
- 需要可用的模型授权

如果不提前讲清楚，用户会误以为只要双击安装器就能直接开始聊天。

## 2. 你要在文档里说清楚什么

至少要明确写出：

- 飞书 `App ID / App Secret` 是在哪里拿到的
- 如果用户使用 `OpenAI Business / Codex OAuth`，不是额外填 API Key，而是先完成 OpenClaw 的登录或配置
- 如果用户使用 API 模式，则要自己准备模型服务的 API 凭证
- 安装完成后还要做一次飞书配对

## 3. 用户自己会操作的环节

新手用户通常要自己完成：

1. 在飞书开放平台创建应用
2. 拿到 `App ID / App Secret`
3. 确认模型授权可用
4. 运行安装器
5. 在飞书中获取配对码
6. 回到安装器点击 `批准配对`

## 4. 最容易卡住的地方

- 不知道飞书 `App ID / App Secret` 去哪拿
- 不知道模型授权不是这个安装器替他创建的
- 不知道安装后还要做配对
- 不清楚第一次运行可能会出现 UAC 提示

## 5. 最推荐的话术

发布时建议直接告诉用户：

`你需要先准备飞书 App ID / Secret，以及一个已经能用的 OpenClaw 模型授权。安装器负责本机安装、飞书配置和配对流程，不负责替你创建飞书应用或模型账号。`
