# 发布前检查清单

如果你准备把这个项目发给新手用户，建议先检查下面这些点。

## 1. 用户需要什么

你要提前告诉用户：

- 需要 `飞书 App ID`
- 需要 `飞书 App Secret`
- 需要先把 OpenClaw 的模型接入跑通
- 需要先把 OpenClaw 的默认模型选好

如果不提前讲清楚，用户会误以为只要双击安装器就能直接开始聊天。

## 2. 你要在文档里说清楚什么

至少要明确写出：

- 飞书 `App ID / App Secret` 是在哪里拿到的
- 模型这一步不只是 `Business` 或 `API` 的区别，而是“模型接入”和“模型选择”两层
- 如果用户使用 `Codex OAuth`，不是额外填 API Key，而是先完成 OpenClaw 的登录或配置，再选默认模型
- 如果用户使用 API 模式，则要自己准备模型服务的 API 凭证，再选默认模型
- 这个安装器只负责飞书接入，不负责替用户完成模型接入或模型选择
- 安装完成后还要做一次飞书配对

## 3. 用户自己会操作的环节

新手用户通常要自己完成：

1. 在飞书开放平台创建应用
2. 拿到 `App ID / App Secret`
3. 在 OpenClaw 里完成模型接入
4. 在 OpenClaw 里选好默认模型
5. 运行安装器
6. 在飞书中获取配对码
7. 回到安装器点击 `批准配对`

## 4. 最容易卡住的地方

- 不知道飞书 `App ID / App Secret` 去哪拿
- 不知道模型接入不是这个安装器替他完成的
- 不知道默认模型也不是这个安装器替他选的
- 不知道安装后还要做配对
- 不清楚第一次运行可能会出现 UAC 提示

## 5. 最推荐的话术

发布时建议直接告诉用户：

`你需要先准备飞书 App ID / Secret，并先在 OpenClaw 里把模型接入和默认模型都配置好。这个安装器负责 Windows 本机安装、飞书配置和配对流程，不负责替你创建飞书应用、模型账号或默认模型。`
