# 发布前检查清单

如果你准备把这个项目发给新手用户，先检查下面这些点。

## 1. 先讲清楚用户要准备什么

至少要提前告诉用户：

1. 需要先按飞书官方文档创建飞书应用
2. 需要拿到 `飞书 App ID`
3. 需要拿到 `飞书 App Secret`
4. 需要先把 OpenClaw 的模型接入跑通
5. 需要先选好 OpenClaw 的默认模型
6. 安装完成后还要做一次飞书配对

## 2. 先讲清楚顺序

推荐顺序必须写清楚：

1. 先准备飞书应用
2. 再准备 OpenClaw 模型
3. 再运行安装器
4. 最后完成配对

如果顺序不写清楚，用户很容易误以为双击安装器就能直接开始聊天。

## 3. 先讲清楚安装器负责什么

安装器负责：

- Windows 上安装和检查依赖
- 写入 OpenClaw 的飞书配置
- 启动网关
- 帮用户完成配对流程

安装器不负责：

- 创建飞书应用
- 替用户完成模型接入
- 替用户选择默认模型

## 4. 最容易卡住的地方

发布前要确认文档里已经明确写到：

1. 飞书 `App ID / App Secret` 去哪里拿
2. 模型为什么要先配
3. 默认模型为什么也要先选
4. 为什么最后还要做一次配对

## 5. 最推荐的对外说明

可以直接对用户说：

`先按飞书官方文档把飞书应用准备好，再把 OpenClaw 模型配好，然后双击 install.cmd，最后按界面完成配对。`
